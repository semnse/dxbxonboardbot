## 🎯 ЧТО УМЕЕТ БОТ

✅ **Привязка карточек Bitrix24 к Telegram чатам**
- Команда `/add <ID>` привязывает карточку из Bitrix24 к чату
- Бот получает данные из Bitrix24 API

✅ **Автоматические напоминания в 9:00 МСК**
- Каждое утро бот отправляет отчёты по всем привязанным карточкам
- Проверяет стадию "Ждём действий клиента"

✅ **Ручные отчёты по команде**
- Команда `/report` показывает текущий статус по карточке

✅ **Интеграция с Bitrix24**
- Получает данные о торговых точках
- Определяет стадию, продукты, причины ожидания

## 📱 ИСПОЛЬЗОВАНИЕ

### 1. Добавьте бота в чат

1. Создайте групповой чат в Telegram с клиентом
2. Добавьте бота **@your_bot_name** как участника
3. Дайте права на отправку сообщений

### 2. Привяжите карточку Bitrix24

1. Откройте карточку в Bitrix24 (Смарт-процессы → Торговые точки)
2. Скопируйте ID из URL:
   ```
   https://your-company.bitrix24.ru/.../crm/leader/1070/6784/
                                                           ^^^^
                                                           ID
   ```
3. В Telegram чате напишите:
   ```
   /add 6784
   ```

4. Бот ответит:
   ```
   ✅ Карточка привязана!
   
   Компания: ООО "Альфа" 7842218770
   ID Bitrix: 6784
   Стадия: Ждём действий клиента - Пауза до вывода
   
   Я буду отправлять отчёты в этот чат каждое утро в 9:00 МСК.
   ```

### 3. Получите отчёт

Напишите в чат:
```
/report
```

Бот отправит персонализированный отчёт.

### 4. Автоматические напоминания

Каждое утро в **9:00 МСК** бот автоматически отправит отчёты по всем активным карточкам.

## ⚠️ ОГРАНИЧЕНИЯ

### Критические:

1. **База данных нестабильна на Windows**
   - Проблема: `asyncpg.exceptions.ConnectionDoesNotExistError`
   - Решение: Использовать синхронный psycopg2 (в разработке)
   - **Обходной путь:** Бот использует кэш в памяти для привязок

2. **Кэш сбрасывается при перезапуске**
   - Проблема: Привязки `/add` хранятся в памяти
   - Решение: После перезапуска приложения нужно выполнить `/add` повторно
   - **Обходной путь:** Сохраняйте ID карточек и перевыполняйте `/add` после рестарта

3. **PostgreSQL должен быть запущен**
   - Проблема: Без БД некоторые функции не работают
   - Решение: Проверьте `sc query postgresql-x64-15`

### Некритические:

4. **UF-поля Bitrix24 не всегда возвращаются**
   - Проблема: Вебхук не имеет прав на чтение пользовательских полей
   - Решение: Настройте права вебхука в Bitrix24

5. **Приватные Telegram каналы не резолвятся**
   - Проблема: Нельзя получить chat_id по invite link
   - Решение: Используйте числовой chat_id или добавьте бота в канал

## 🔧 РЕШЕНИЕ ПРОБЛЕМ

### Проблема: Бот не отвечает на команды

**Решение:**
1. Проверьте, запущено ли приложение:
   ```bash
   curl http://localhost:8000/
   ```
2. Проверьте логи:
   ```bash
   type logs\uvicorn.log
   ```
3. Перезапустите приложение:
   ```bash
   taskkill /F /IM python.exe
   python -m uvicorn app.main:app --reload
   ```

### Проблема: Ошибка "connection was closed in the middle of operation"

**Решение:**
1. Проверьте PostgreSQL:
   ```bash
   sc query postgresql-x64-15
   ```
2. Если остановлен — запустите от администратора:
   ```bash
   net start postgresql-x64-15
   ```
3. Если не помогает — перевыполните `/add <ID>`

### Проблема: Бот не получает сообщения

**Решение:**
1. Убедитесь, что бот добавлен в чат
2. Проверьте права бота (должен иметь право на чтение сообщений)
3. Удалите и добавьте бота снова

### Проблема: Карточка не находится в Bitrix24

**Решение:**
1. Проверьте ID карточки (должен быть числом)
2. Убедитесь, что карточка существует в Bitrix24
3. Проверьте права вебхука Bitrix24
