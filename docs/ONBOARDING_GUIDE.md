## Telegram Bot DocsInBox

**Для кого:** Внедренцы, менеджеры проектов, руководители

---

## 🎯 Назначение бота

Бот автоматически отправляет **ежедневные отчёты** о статусе внедрения в Telegram-чаты клиентов.

### Что умеет:
- ✅ Отслеживает статусы сделок в Bitrix24
- ✅ Отправляет отчёты каждое утро в 9:00 МСК
- ✅ Работает в обычных чатах и чатах с топиками (Topics)
- ✅ Привязывается к конкретным карточкам Bitrix24
- ✅ Показывает **доступные действия** для каждого продукта
- ✅ Формирует отчёты по продуктам (ЕГАИС, Меркурий, Маркировка, Накладные, ЮЗЭДО)
- ✅ Показывает **риски** и почему важно выполнить действия

---

## 🚀 Быстрый старт

### Шаг 1. Создать чат с клиентом

**Вариант A: Обычный чат (без топиков)**
1. Создайте группу в Telegram
2. Добавьте клиента и команду проекта
3. Добавьте бота `@docsinbox_onboardbot`

**Вариант B: Чат с топиками (рекомендуется)**
1. Создайте группу в Telegram
2. Включите режим **"Форум"** (Topics)
3. Создайте топики по клиентам:
   - `ООО Ромашка`
   - `ООО Восход`
   - `ООО Фортуна`
4. Добавьте бота `@docsinbox_onboardbot`

---

### Шаг 2. Привязать карточку Bitrix24

1. Откройте карточку сделки в Bitrix24
2. Скопируйте ID карточки из URL:
   ```
   https://docsinbox.bitrix24.ru/crm/leader/1070/12345/
   ↑ ID карточки = 12345
   ```
3. В Telegram-чате (или топике) напишите:
   ```
   /add 12345
   ```
4. Бот подтвердит привязку:
   ```
   ✅ Карточка привязана!

   Компания: ООО Ромашка
   Стадия: Ждём действий клиента
   ```

---

### Шаг 3. Получать отчёты

**Ежедневно в 9:00 МСК** бот автоматически отправит отчёт:

```
🔍 Отчёт для ООО "Ромашка"

📋 Данные из Bitrix24:
• ИНН: 5024076921
• Стадия: Ждём действий клиента

✅ Подключённые продукты:
• ЕГАИС
• Меркурий
• Накладные

⏳ Осталось сделать:
• Провести сопоставления номенклатур
• Заполнить торговый зал
• Подтвердить легальность площадки в Меркурии

💡 Это важно, потому что:
Без этих шагов вы не сможете передавать чеки в ЕГАИС. 
Это может привести к штрафам от Росалкогольрегулирования.

✅ Доступно на этой стадии:
• Меркурий:
  - Получение, гашение и возврат ВСД
• Накладные:
  - Получение, приемка накладных

✨ Docsinbox Внедрение — ваш надёжный помощник!
```

**По команде `/report`** вы получите такой же отчёт в любое время!

## 🏢 Сценарии использования

### Сценарий 1: Один клиент = один чат

**Когда использовать:** Для крупных клиентов с индивидуальным подходом

```
Чат: "ООО Ромашка | Внедрение"
Участники: 
  - Менеджер (вы)
  - Клиент (ЛПР)
  - Технический специалист

Команда: /add 12345
Отчёты: Каждое утро в 9:00 в этот чат
```

---

### Сценарий 2: Несколько клиентов = один чат с топиками ⭐

**Когда использовать:** Для массового внедрения (10+ клиентов)

```
Группа: "Внедрение клиентов | Март 2026"
Топики:
  ├─ Топик 1: "ООО Ромашка" → /add 12345
  ├─ Топик 2: "ООО Восход" → /add 67890
  ├─ Топик 3: "ООО Фортуна" → /add 11111
  └─ Топик 4: "General" (без привязки)

Отчёты: Каждое утро в 9:00 в соответствующий топик
```

**Преимущества:**
- ✅ Все клиенты в одном месте
- ✅ Отчёты приходят в нужные топики
- ✅ Удобно контролировать多个 клиентов

---

### Сценарий 3: Внутренний чат отдела

**Когда использовать:** Для контроля всех внедрений отдела

```
Чат: "Отдел внедрения | Мониторинг"
Участники:
  - Руководитель отдела
  - Все внедренцы
  - Бот

Команды: /add <ID> для каждого клиента
Отчёты: Статус по всем клиентам в одном чате
```

## 🔧 Логика работы бота

### Как бот понимает куда отправлять отчёты?

```
1. Вы пишете /add 12345 в топике "ООО Ромашка"
   ↓
2. Бот сохраняет привязку в БД:
   - chat_id: -1003876900857 (ID группы)
   - message_thread_id: 2 (ID топика)
   - bitrix_deal_id: 12345 (ID карточки)
   ↓
3. В 9:00 MSK Celery задача:
   - Читает привязки из БД
   - Получает данные из Bitrix24
   - Отправляет в chat_id + message_thread_id
   ↓
4. Отчёт приходит в топик "ООО Ромашка"
```

---

### Что происходит при перезапуске бота?

```
✅ Привязки сохраняются в PostgreSQL
✅ При рестарте бот читает привязки из БД
✅ Отчёты продолжают приходить
```

**Важно:** Привязки **не теряются** при перезапуске!

## ⚠️ Частые проблемы и решения

### Проблема 1: Бот не отправляет отчёты

**Причина:** Бот заблокирован в чате

**Решение:**
1. Проверьте что бот в чате
2. Убедитесь что бот не заблокирован
3. Проверьте привязку: `/report`

---

### Проблема 2: Отчёты приходят не в тот топик

**Причина:** Привязка сделана в другом топике

**Решение:**
1. Проверьте в каком топике делали `/add`
2. Сделайте `/add <ID>` в правильном топике
3. Старая привязка перезапишется

---

### Проблема 3: Бот не видит карточку

**Причина:** Неверный ID карточки

**Решение:**
1. Проверьте ID в URL Bitrix24
2. Убедитесь что карточка существует
3. Попробуйте `/add <ID>` снова

---

### Проблема 4: "К этому чату не привязана карточка"

**Причина:** Привязка не найдена в БД

**Решение:**
```
/add <ID карточки из Bitrix24>
```

## 🎓 Лучшие практики

### ✅ Делать:

1. **Создавайте отдельный топик для каждого клиента**
   - Удобно отслеживать историю
   - Отчёты приходят в нужный топик

2. **Добавляйте ЛПР клиента в чат**
   - Клиент видит отчёты
   - Меньше вопросов "как дела?"

3. **Проверяйте привязку после /add**
   - Убедитесь что бот ответил "✅ Карточка привязана"
   - Проверьте `/report` для теста

4. **Используйте Topics для массового внедрения**
   - 10+ клиентов в одном чате
   - Каждый в своём топике

---

### ❌ Не делать:

1. **Не удаляйте бота из чата**
   - Сбивается привязка
   - Отчёты перестают приходить

2. **Не меняйте название топика после привязки**
   - Бот продолжит отправлять (по chat_id)
   - Но может быть путаница

3. **Не делайте /add в General**
   - Отчёты будут приходить в General
   - А не в топик клиента
